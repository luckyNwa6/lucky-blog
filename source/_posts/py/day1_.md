# 机器学习(Machine Learning)

**规律**

2、4、6、8

y=2x

当 x = 5 → y = 10 这就是 **预测**

模型 = 函数 ；预测 = 函数把 x 映射到 y

函数也可以说是一种模型来预测

### 机器学习的应用

花朵数据（大小、颜色） → 花的种类

健康指标（血压血脂） → 是否患病

房屋信息（面积、位置） → 房价

任务的共同点：就是下面有监督学习那 4 点

### 有监督学习

“有监督” = 有标签

1、需要一定数量的训练样本 面积、位置 → 房价（标签）

2、输入和输出之间有关联关系 如果没有规律，再多数据也学不到

3、输入和输出可以数值化表示 模型只能处理数字，图片、文本、声音都要转成数字

4、任务需要有预测价值 预测“你的鞋码是不是奇数” ，没有意义

**应用**

文本分类任务 输入--->文本 输出--->类别 关系：文本的内容决定了文本的类别

机器翻译任务、图像识别任务（输入图输出类别）、语言识别任务

### **无监督学习**

无监督学习 = 只有输入 X，没有标签 y

它的目标是**分析数据本身的结构**

给与机器的数据没有标注信息，通过算法对数据进行一定的自动分析处理，得到一些结论

比如：聚类、降维、找特征值等

无监督没有“正确答案”，模型只做数据分析

### **流程**

训练数据----->数据处理----->算法选择----->建模&评估----->算法调优----->模型（公式|函数）

1. 训练数据 ：收集数据、清洗。

2. 数据处理：归一化、标准化、文本分词、图像缩放等。

3. 算法选择：机器学习选择：SVM、XGBoost 等 深度学习选择：CNN、Transformer 等

4. 建模 & 评估：模型训练完后，需要验证效果。

5. 算法调优：调参、调整结构、换优化器等。

6. 得到最终模型（函数）：最终输出的就是一个模型（函数）

### 数据集

#### **训练集**

用于模型训练的训练数据集合，让模型学习规律

#### **验证集**

用来“选模型”和“调参数”

例如：

- 哪个模型结构更好？
- 学习率多大合适？
- 多少层神经网络效果最好？

验证集不参与训练，只用于对比和调优

对于每种任务一般都有多种算法可以选择，一般会用验证集验证用于对比不同算法的效果差异

#### **测试集**

最终评分使用
它模拟真实环境

要求：

- 不能参与训练
- 不能参与调参
- 必须全新
- 只能在训练完后用一次

最终用于评判算法模型效果的数据集合

### **K 折交叉验证**

把数据分成 K 份：

```
Fold1 Fold2 Fold3 ... FoldK
```

每次：

- 其中 1 份作为验证集
- 其他 K-1 份作为训练集

重复 K 次，最后取平均值。

优点：

- 数据少时非常有效
- 评估更稳定

初始采样分割成 K 个子样本，一个单独的子样本被保留作为验证模型的数据，其他 K-1 个样本用来训练。交叉验证重复 K 次，每个子样本验证一次，平均 K 次的结果

### **评价指标**

主要用于分类任务：

1. **准确率 Accuracy**
2. **召回率 Recall**
3. **F1-score**
4. **Top-K Accuracy（常用于图像分类）**

不同任务使用不同指标

### 模型效果不好

#### **过拟合**

定义：

- 在训练集表现非常好
- 在验证集和测试集表现非常差

原因：

- 模型记住训练数据
- 没有学到泛化规律

就像学生只会背题，不会做新题

模型失去了范化能力，如果模型再训练集和验证集上有很好表现，但在测试集神表现很差，一般认为发生了过拟合

#### **欠拟合**

定义：

- 在训练集表现就很差
- 表示模型太弱，无法学习规律

原因：

- 模型太简单（例如用线性模型学很复杂的曲线）
- 数据特征不足

就像学生数学基础差，连训练题都不会做

模型没能建立起合理的输入输出之间的映射，当输入训练集中的样本时，预测结果和标注结果依然相差很大

# 深度学习(Deep Learning)

深度学习是一种“利用多层神经网络”的机器学习方法

它不仅仅包含模型结构，还包含：

- 神经网络结构设计（CNN、RNN、Transformer……）
- 优化器（SGD、Adam）
- 损失函数（交叉熵、MSE…）
- 梯度下降
- 正则化、Dropout
- 数据增强
- 训练策略
- 大模型预训练、微调
- 框架（PyTorch、TensorFlow）

深度学习 = 神经网络 + 整套训练方法 + 实践体系

**深度学习的基础思想：**

深度学习的整个流程只为了：让模型输出更靠近真实答案

选公式--->参数随机初始化--->标注数据算误差--->根据误差调整参数简而言之，“先猜后调”

怎么做到？

预测 → 衡量误差 → 求导 → 下降

我们有一堆数据，希望模型能够从数据中学习规律，并在未来对**未知样本做预测**。

深度学习的关键思想：

> 用一堆可学习的参数（权重）构建复杂函数，
>
> 让它自动拟合数据，而不是人工设计规则。

猜数字

A:我心理想了一个整数 0-100，你猜

B：60 A：低了 B：80 A：低了

B：90 A：高了 B：88 A：正确

```
首先B随便猜一个数                   ---模型随机初始化
	模型函数:Y=k*x (此样本x=100)
	此例子中B选择的初始值K值为0.6
A计算B的猜测与真正答案的差距          ---计算loss
	损失函数=sign(y_true-y_pred)
A告诉B偏大或偏小                    ---得到loss值
B调整了自己的“模型参数”              ---反向传播
参数调整幅度按照B自定的策略           ---优化器&学习率
重复以上过程
最终B的猜测与A的答案一直             ---loss=0
```

### **损失函数**

损失函数（loss function）用来计算模型的预测值与真实值之间的误差

模型训练的目标一般是依靠训练数据来调整模型参数，使得损失函数到达最小值

损失函数有很多，选择合理的损失函数是模型训练的必要条件

### **倒数与梯度**

导数表示函数曲线上的切线斜率。除了切线的斜率，导数还能表示函数在该点的变化率

### **梯度下降**

梯度是损失函数对参数的导数，告诉我们函数向哪个方向增长最快，那么他的反方向，就是下降最快的方法

梯度指向损失上升方向，我们要朝损失下降的方向走，所以乘以 -1

梯度下降的目的是找到函数的极小值

为什么要找函数的极小值？因为我们的最终目的是损失函数值最小

### **优化器**

一种用梯度来更新参数的优化算法

知道走的方向，还需要知道走多远

假如一步走太大，就可能错过最小值，如果一步太小，又可能困在某个局部低点无法离开

学习率（learning rate），动量（Momentum）都是优化器相关的概念

### **Batch&Mini Batch & Epoch**

**Batch(一次批次)** ：模型一次看到的数据量，但通常 Batch 太大，显存不够

**Mini Batch(小批次)** ：把训练集拆成很多小份，每次从中取一小份训练模型

- 数据集：10000 个样本
- Mini-Batch 和 batch size：100

那么训练时会这样：一次训练用 100 个样本 → 训练完一次叫做 1 step 或 1 iteration

**Epoch(一个轮次)**

一个 epoch = 模型把整个训练集“完整看一遍”。

继续刚才的例子：

- 数据集：10000 个样本
- Batch size：100

每个 epoch 需要 10000 / 100=100 次训练步骤

也就是说：100 次 mini-batch 更新 = 1 个 epoch

一般 batch size 用：34,64,128,256,512

### 神经网络（Neural Network）

神经网络（NN）是一个**数学结构**，主要由以下组成：

\* 输入层

\* 多个隐含层

\* 输出层

\* 权重

\* 激活函数（relu、sigmoid、tanh）

\* 前向传播

\* 反向传播

\* 损失函数

这本质上是：

$$
{输出} = f(Wx + b)
$$

一个最简单的三层网络就是：

输入 → 隐层 → 输出

**隐含层/中间层** : 神经网络模型输入层和输出层之间的部分

隐含层可以有不同的结构：

- **DNN**：普通全连接层（矩阵乘法）
- **CNN**：卷积运算（提取局部模式）
- **RNN**：循环结构（时间序列）
- **LSTM**：门机制（记忆能力）
- **Transformer**：自注意力机制

底层本质：把输入通过某种数学方式变成输出

例如 DNN： h=Wx+b

**随机初始化**

隐含层会含有很多的权重矩阵，这些矩阵需要有初始值，才能进行运算

初始值的选取会影响最终的结果

一般情况下，模型会采取随机初始化，但参数会在一定范围内

在使用预训练模型一类的策略时，随机初始值被训练好的参数代替

**训练神经网络的整个目标**：找到一组参数，让模型输出尽量接近真实值

为什么要：

| 步骤     | 目的         |
| -------- | ------------ |
| 预测     | 看当前能力   |
| 损失函数 | 衡量错误     |
| 导数     | 找到改进方向 |
| 梯度下降 | 调整参数     |
| 反复训练 | 逐渐优化     |

这不是程序员想出来的流程， 这是数学推导出的**最有效方法**

模型预测本身**不需要导数**，但为了让模型预测更准，我们需要：

1. 计算预测和真实答案的差距（loss）
2. 反向传播计算导数（gradient）
3. 用导数更新参数（update parameters）

导数是用来改进预测，而不是用来做预测
